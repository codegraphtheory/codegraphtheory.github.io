why this exists

The useful part, not a generated placeholder.

who it is for

Builders who want a specialist AI profile repo that can be installed, validated, released, and customized without starting from a blank prompt.

problem

It turns profile creation into an auditable software artifact. The repo ships manifests, SOUL.md, skills, examples, release checks, and CI instead of a disposable chat transcript.

positioning

Original GraphTheory work with source, docs, repository metadata, and a direct path back to GitHub for issues and contributions.

workflow

How the project is meant to be used or evaluated.

01Start from one sentence or a structured params file.
02Generate a complete profile distribution with deterministic scripts.
03Validate manifests, skills, examples, runtime ownership, and install paths.
04Publish with README copy, catalog snippets, release readiness, and smoke tests.

quick start

First command path. Check the README for current details before production use.

hermes profile install github.com/codegraphtheory/hermes-profile-template --name profile-architect --alias profile-architect chat
